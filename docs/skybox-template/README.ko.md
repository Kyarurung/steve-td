# SemionTD 스카이박스 템플릿

이 템플릿은 SemionTD의 `config/semion-td/skyboxes/`에 넣을 큐브맵 PNG를 만들 때 사용합니다. 모델과 fog 우회 셰이더는 서버가 생성 리소스팩에 자동으로 추가하므로 별도 리소스팩을 만들 필요가 없습니다.

Ryan Remer의 무료 네뷸라 자산으로 만든 4종의 출처와 등록 ID는 [Ryan Remer 네뷸라 스카이박스](RYANREMER-NEBULAS.ko.md)에서 확인합니다.

## 파일

- `skybox-layout.svg`: 1024×512 편집 원본입니다. `artwork` 그룹에 그림을 넣고 `guides` 그룹을 숨긴 뒤 PNG로 내보냅니다.
- `skybox-layout.png`: 얼굴 위치를 빠르게 확인하는 참고 이미지입니다. 이 파일은 안내 문자와 색상이 들어 있어 실제 스카이박스로 쓰지 않습니다.

## 얼굴 배치

아틀라스는 256×256 칸 4개를 가로로, 2개를 세로로 배치한 1024×512 이미지입니다.

```text
┌────────┬────────┬────────┬────────┐
│ 미사용 │   UP   │  DOWN  │ 미사용 │
├────────┼────────┼────────┼────────┤
│  WEST  │ SOUTH  │  EAST  │ NORTH  │
└────────┴────────┴────────┴────────┘
```

`DOWN` 칸은 모델에서 세로 방향이 반전됩니다. 바닥에 글자나 방향성이 강한 무늬를 넣었다면 게임 안에서 방향을 확인합니다.

## 만드는 순서

1. `skybox-layout.svg`를 Inkscape, Illustrator, Figma 같은 편집기에서 엽니다.
2. `artwork` 그룹에 여섯 면의 이미지를 배치합니다. 사용되는 각 칸은 빈틈 없이 채웁니다.
3. `guides` 그룹을 숨기거나 삭제합니다.
4. 캔버스 크기를 유지한 채 1024×512 PNG로 내보냅니다.
5. 파일명을 영문 소문자, 숫자, `_`, `-`로 정합니다. 예: `red_nebula.png`.
6. PNG를 서버의 `config/semion-td/skyboxes/`에 넣고 `/semiontd resourcepack reload`를 실행합니다.
7. `/스카이박스`로 등록 여부를 확인하고 `/스카이박스 red_nebula`로 선택합니다.

2048×1024처럼 같은 2:1 비율로 확대해도 됩니다. 얼굴 순서와 각 칸의 상대 위치는 바꾸지 않습니다.

## Alpha와 fog

PNG의 불투명 픽셀은 생성 리소스팩에 들어갈 때 alpha `252`로 자동 변환됩니다. 이 alpha를 표식으로 사용해 스카이박스에만 fog 적용을 건너뜁니다. 편집 단계에서 alpha를 직접 `252`로 맞출 필요는 없습니다.

완전 투명한 픽셀은 그대로 투명하게 남습니다. 실제 얼굴 영역에 투명한 구멍이 있으면 바닐라 하늘이 보일 수 있습니다.
