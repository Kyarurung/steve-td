# 범위 효과와 AOE VFX 구현 기록

**작성일:** 2026-07-10

**상태:** 구현 완료

## 구현 결과

피해 AOE, 시체 폭발, 버프·디버프, 회복 오라를 `AreaEffectApi`로 통합했습니다. 타워별 코드는 범위 검색을 직접 하지 않고 효과 요청과 대상별 액션만 정의합니다.

```text
타워 능력
-> AreaEffectApi 요청
-> 메인 스레드에서 라인·거리·필터 검사
-> 대상별 액션 실행
-> AreaEffectResult와 적용 전 위치 스냅샷 생성
-> VFX 이벤트 큐
-> 단일 VFX 작업 스레드에서 스타일 계획과 전송
```

공개 사용법은 [범위 효과 API](../area-effect-api.ko.md)에 정리했습니다.

## 메인 스레드 경계

메인 스레드에서 처리하는 작업:

- 활성 경기와 소스 타워의 라인 확인
- Semion 몬스터·타워 검색
- 구형 거리와 추가 필터 검사
- 피해, 회복, 버프·디버프 적용
- 후보 수, 적용 수, 처치 수 집계
- 적용 전 위치와 VFX 수신자 스냅샷 생성

VFX 작업 스레드에서 처리하는 작업:

- `Line`, `Circle`, `Sphere`, `Trail` 도형 계획
- APEL point 계산
- 바닐라 particle packet과 GCB payload 생성
- 라인별 예산 차감과 연결 전송 큐 등록

작업 스레드는 월드, 타워, 몬스터 상태를 읽지 않습니다.

## 대상 범위

몬스터 효과는 살아 있는 Semion 몬스터와 소스 타워가 방어하는 라인만 대상으로 합니다. 타워 효과는 같은 소유자·팀·라인의 Semion 타워와 요청한 환영만 대상으로 합니다.

타워 대상 모드는 다음과 같습니다.

- `REGISTERED`: 라인에 등록된 런타임 타워
- `ENTITIES`: 월드의 Semion 타워 엔티티
- `REGISTERED_AND_CLONES`: 등록 타워와 환영 엔티티

공격 대상 선택, 기본 공격 사거리, 몬스터·소환수 능력, 주변 사망 스택 판정은 범위 효과가 아니므로 기존 경로를 유지합니다.

## 피해와 처치

공통 피해 경로는 `TowerAreaDamage`가 `AreaEffectApi` 위에서 처리합니다. 대상별 피해량과 피해 후 흡혈 같은 후처리를 전달할 수 있습니다.

처치 효과 전파 여부는 호출 경로가 정합니다. 일반 스플래시는 처치 시 `Tower.onKill(...)`을 호출하고, 라클 캣 시체 폭발은 호출하지 않아 연쇄 폭발을 막습니다.

이관된 피해 경로:

- 공통 스플래시와 주민 계열 스플래시·가시·라클 캣
- 동물 돼지·늑대
- 흑마법사 스플래시
- 무리 범위 피해
- 우민 범위 피해
- 언데드 허스크 pulse
- 공명 범위 피해
- 네더 대상 중심 스플래시와 타워 중심 pulse

## 버프·디버프·회복

비피해 효과는 `ON_CHANGE` 정책을 사용합니다. 새 효과나 수치 변화가 있을 때만 VFX를 만들고, 지속 시간만 갱신되면 `UNCHANGED`를 반환합니다.

이관된 경로:

- 알레이 회복·무기 강화·방어 강화
- 염소 일반 타워·환영 강화
- 공명 보호와 피해 동반 디버프
- 언데드 동물 공격력 감소·받는 타워 피해 증가
- 흑마법사 희생 타워 사망 디버프

공명 피해와 디버프처럼 같은 대상을 쓰는 능력은 한 번 검색한 결과에 두 효과를 적용합니다.

## VFX 스타일

내장 스타일:

- `semion-td:splash`
- `semion-td:pulse`
- `semion-td:corpse_explosion`
- `semion-td:buff`
- `semion-td:debuff`

피해 AOE와 시체 폭발은 대상이 없어도 `ON_TRIGGER`로 외곽선을 표시합니다. 버프·디버프·회복은 `ON_CHANGE`를 사용합니다. 밝은 바닐라 `FLASH` 파티클은 사용하지 않습니다.

애드온은 `SemionTdApi.areaVfxStyles()`에 스타일을 등록할 수 있습니다. 등록은 서버 시작 전에 끝나야 하며, 플래너는 불변 컨텍스트와 제한된 도형 출력만 받습니다.

## 예산과 수신자

`vfx.json`의 기존 기본 예산을 유지합니다.

- 일반 클라이언트: 라인별 tick당 `1024` point, burst `4096` point
- 일반 클라이언트 수신자별 tick당 최대 `2048` packet
- GCB 클라이언트: 라인별 tick당 최대 `128` 도형 명령
- 라인 큐: 최대 `512` 이벤트
- 지연 허용: 최대 `1 tick`, `2 tick` 이상 지난 이벤트는 폐기

타워 소유자와 같은 월드에서 효과 중심 64블록 안의 경기 관전자에게만 전송합니다. 다른 라인 참가자는 수신자에서 제외합니다.

## 검증 기준

- 다른 팀과 라인의 대상이 효과 후보에 들어가지 않습니다.
- 요청 반경과 실제 적용 대상 위치가 VFX 컨텍스트에 반영됩니다.
- 라클 캣 시체 폭발이 연쇄 폭발을 만들지 않습니다.
- 환영 포함 여부가 타워 대상 모드에 맞습니다.
- 버프·디버프 지속 시간 갱신만으로 VFX가 반복되지 않습니다.
- 외부 플래너 실패가 서버 tick과 다른 VFX를 중단하지 않습니다.
- 모든 스타일이 라인별 point와 GCB 명령 예산을 지킵니다.
